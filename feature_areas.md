# Cosmic AI Backend - Feature Areas

Here is a comprehensive list of all the feature areas present in the `cosmic-ai-backend` project, based on the application's routes and models.

| Feature Area | Description | Related Files (Routes & Models) |
| :--- | :--- | :--- |
| **Authentication & Users** | Handling user registration, login, profile management, and authentication flows. | `authRoutes.js`, `userRoutes.js`, `User.js` |
| **AI Chat** | Conversational AI interface for personalized astrological and spiritual guidance. | `aiChatRoutes.js`, `AiChat.js` |
| **Astrological Charts** | Generation and management of birth charts (natal charts) and planetary placements. | `chartRoutes.js`, `BirthChart.js` |
| **Community & Social** | Social features including forums, posts, and user interactions. | `communityRoutes.js`, `Community.js` |
| **Compatibility (Synastry)** | Relationship compatibility analysis between different users or charts. | `compatibilityRoutes.js`, `Compatibility.js` |
| **Horoscopes** | Delivery of daily, weekly, or monthly personalized horoscopes. | `horoscopeRoutes.js` |
| **Journaling** | A private space for users to log their thoughts, reflections, and spiritual journey. | `journalRoutes.js`, `JournalEntry.js` |
| **Learning Module** | Educational content and resources related to astrology, tarot, etc. | `learnRoutes.js`, `Learn.js` |
| **Tarot Readings** | Functionality related to drawing and interpreting Tarot cards. | `readingsRoutes.js` |
| **Calendar & Transits** | Tracking celestial events, moon phases, and personal planetary transits. | `calendarRoutes.js` |
| **Location Services (City)** | Geographic lookups required for accurate birth chart generation. | `cityRoutes.js`, `City.js` |
| **Billing & Subscriptions** | Handling premium features, payments, and subscription tiers. | `billingRoutes.js` |
| **Notifications** | Sending push notifications and in-app alerts to users. | `notificationRoutes.js` |
| **Settings & Preferences** | User-specific application configuration and preferences. | `settingsRoutes.js` |
| **Media & Uploads** | Management of file and media uploads (e.g., profile pictures). | `uploadRoutes.js`, `Upload.js` |
| **Content Management** | Core system for handling static or dynamic content entries. | `Content.js` |
